The Challenge

Solo travellers, bikers, and backpackers often struggle to plan and organise trips with detailed itineraries, budgets, and packing lists. They also lack a reliable way to find nearby fellow travellers for safety and companionship.

Existing travel applications are either commercial, expensive, or do not cater to the specific needs of solo adventure travellers who prefer collaborative, lightweight trip management.

Our Solution

Roamly is a community-driven mobile travel application that helps users plan trips collaboratively and discover fellow travellers on a live interactive map.

  • Full trip planning with locations, hotels, budgets, and packing lists
  • Real-time companion discovery via GPS-based location sharing
  • Collaborative trips with companion and editor invitations
  • Built entirely on free-tier services (zero operational cost)

Features Implemented
Core services and functionalities from the syllabus
User Authentication
Sign up with name, email, phone, and password. Google Sign-In as an alternative. Role-based routing (user vs. admin).
Interactive Map (GPS Service)
Full-screen Mapbox map with real-time GPS blue-dot positioning. Runtime location permission handling with user feedback.
Trip Planning & Management
Create detailed trip itineraries with multi-stop locations, hotel bookings, budget tracking, and packing checklists. Status tracking (planned, active, completed).
Location Search (Network Service)
Two-stage Mapbox Geocoding API integration (suggest and retrieve). Map picker for selecting trip destinations and stops.
Companion Discovery (Real-Time)
Discoverable toggle broadcasts GPS coordinates. Nearby travellers appear as live markers on a shared map with connection requests.
Trip Collaboration
Invite companions (viewers) and editors (collaborators) to trips. Accept or decline invitations with real-time Firestore sync.

GPS coordinates saved to Firestore only when the user moves more than 500 metres, minimising unnecessary database writes.

Application Architecture
Feature-first directory structure following Separation of Concerns

Project Structure

1
core/
Shared infrastructure: services (GPS, geocoding, trip management, user management), themes (light/dark), and configuration constants.
2
features/
Screens grouped by feature: auth, home, trips, companions, profile, and shared widgets.
3
models/
Data classes: UserProfile, TripModel, TripLocation, TripHotel, BudgetItem, PackingItem, CompanionRequest.

Design Principles

  • Separation of Concerns — UI code never calls the database directly; all data access goes through service classes
  • Login-First Architecture — Authentication required before any feature is accessible
  • Role-Based Routing — Admin users redirected to moderation dashboard; regular users to the map
  • Real-Time Data — Firestore streams used for live companion tracking and trip collaboration
  • Zero-Cost Backend — All services operate within free-tier limits

Conclusion
Key takeaways and future roadmap

What We Learned

  • Building a production-quality Flutter application with feature-first architecture and clean separation of concerns
  • Integrating Firebase Authentication with multiple sign-in methods (email/password and Google OAuth)
  • Working with Cloud Firestore real-time streams for live data synchronisation across multiple users
  • Implementing GPS services with proper runtime permission handling and user feedback
  • Consuming third-party REST APIs (Mapbox Geocoding) with session management and rate limiting
  • Designing collaborative data models with nested documents, role-based access (owner, editor, companion), and invitation workflows

Future Improvements

  • In-app messaging — Allow connected companions to chat directly within the application
  • Photo uploads — Allow users to attach trip photos using Firebase Storage
  • Push notifications — Notify users of companion requests and trip invitations via Firebase Cloud Messaging
  • Offline support — Cache trip data locally for areas with limited connectivity
  • Route optimisation — Auto-order trip locations by shortest travel distance using Mapbox Directions API
  • iOS deployment — Extend the application to iOS using Flutter's cross-platform capabilities
